# Predictive Multi-UPF Latency- and Energy-Aware Dynamic Placement

This system is a complete end-to-end framework integrating Python-based control logic with ns-3 network simulation and NetAnim visualization to demonstrate UPF placement optimization.

## Features
- **Python Controller (`main.py`)**: Simulates user traffic and mobility natively, predicting the best UPF placement using standard and predictive algorithms to optimize for latency, energy cost, and SLA violations.
- **Python/ns-3 Pipeline (`run_pipeline.py`)**: Runs Python simulation, exports the topology to `topology_static.json` and `topology_predictive.json`, and invokes the ns-3 C++ simulation for detailed packet-level metrics and FlowMonitor insights.
- **NetAnim Output**: `upf-sim.cc` yields NetAnim XML files highlighting RED nodes as UPFs and BLUE nodes as Users/Routers.

## Execution Requirements
To fully execute the system:
1. Ensure Python dependencies are installed (e.g. `networkx`, `matplotlib`, `numpy`).
2. Move this entire structure into your `ns-3` development directory (e.g. `ns-allinone-3.XX/ns-3.XX/`). 
   - `scratch/upf-sim.cc` must reside in the ns-3 `scratch` directory.
   - Run the script from the ns-3 root directory so `./ns3 run` will resolve correctly.

### 1. Standalone Python Test
To run just the Python algorithms and generate the initial evaluation plots:
```bash
python main.py
```
This generates:
- `python_metrics.png` (Visualizes latency, energy, SLA, and cost over time).
- `topology_static.json` and `topology_predictive.json`.

### 2. Full ns-3 Integration Pipeline
To simulate with full packet-level routing in ns-3:
```bash
python run_pipeline.py
```
This script handles executing ns-3 for both the Static and Predictive topologies. It safely outputs:
- `flowmon_static.xml` / `flowmon_predictive.xml`
- `static.xml` / `predictive.xml` (For NetAnim Visualization)

### 3. NetAnim Visualization
To visualize the generated topologies:
1. Build NetAnim (included with ns-allinone installations).
2. Launch NetAnim natively: `./NetAnim`
3. Load the `static.xml` or `predictive.xml` files generated by the pipeline.
- Observe packet movement.
- RED Nodes = UPF
- BLUE Nodes = Users / Standard Network Routers
